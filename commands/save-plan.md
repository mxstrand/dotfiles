Save a plan or design document to `.claude-docs/plans/YYYY-MM-DD-[brief-topic].md` in current working directory.

## Filename Generation

- Use brief, descriptive topics related to the plan/design (e.g., "feature-auth", "refactor-api", "migration-v2")
- Format: `YYYY-MM-DD-[topic].md`

## Document Structure

- **Repository**: [current repo name/path]
- **Date**: YYYY-MM-DD
- **Objective**: Brief summary of what's being planned
- **Plan**: The specific approach or design
- **Key Files**: Files that will be affected
- **Completion Criteria**: How we'll know it's done

Keep it concise and actionable. This serves as a reference for planned work.

## After Saving

Remind the user: "Plan saved locally. You can also save to https://github.com/mxstrand/claude-plans for cross-Codespace persistence if desired."
